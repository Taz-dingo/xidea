# Demo Showcase Strategy

## Core Narrative

Xidea 要展示的不是若干孤立功能，而是同一个 AI 学习编排系统如何调度不同输入与训练方式。

当前比赛版主案例默认是：
AI 工程师围绕真实项目学习 RAG 系统设计。
这个案例优先用来证明为什么学习任务需要 `project / thread` 和动态动作选择。

## Demo Structure

### 主叙事

1. 用户带着项目目标进入系统
2. 系统理解内容与用户状态
3. 系统选择下一步训练动作
4. 系统解释为什么这样安排
5. 系统把结果回写成后续学习依据

### 可展示的能力 demo

- 原始材料导入
- 普通问答进入并形成画像
- 导师问答
- 苏格拉底追问
- 对比辨析
- 情境模拟
- 视觉或听觉训练

## Showcase Rule

每个 demo 都要回答同一个问题：

- 它解决的学习状态是什么
- 为什么这时候该用这个训练方式
- 它如何体现系统编排，而不是单点功能

每个 demo 默认都要把这 5 个证据点显式展示出来：

- 当前项目目标是什么
- 当前学习状态是什么
- 系统为什么选这个动作
- 当前路径如何安排
- 哪些状态被更新了

## Anti-Pattern

不要把 demo 讲成：

- 我们也能聊天
- 我们也能导入文件
- 我们也能做图片训练
- 我们未来什么学科都能覆盖

要讲成：

- 当用户从不同入口进入时，系统如何统一理解他
- 当用户处于不同状态时，系统如何切换不同训练方式
- 为什么这个学习任务需要围绕项目持续编排，而不是一次性回答
