# Demo Showcase Strategy

## Core Narrative

Xidea 要展示的不是若干孤立功能，而是同一个 AI 学习编排系统如何调度不同输入与训练方式。

## Demo Structure

### 主叙事

1. 用户进入系统
2. 系统理解内容与用户状态
3. 系统选择下一步训练动作
4. 系统解释为什么这样安排

### 可展示的能力 demo

- 原始材料导入
- 普通问答进入并形成画像
- 导师问答
- 苏格拉底追问
- 对比辨析
- 情境模拟
- 视觉或听觉训练

## Showcase Rule

每个 demo 都要回答同一个问题：

- 它解决的学习状态是什么
- 为什么这时候该用这个训练方式
- 它如何体现系统编排，而不是单点功能

## Anti-Pattern

不要把 demo 讲成：

- 我们也能聊天
- 我们也能导入文件
- 我们也能做图片训练

要讲成：

- 当用户从不同入口进入时，系统如何统一理解他
- 当用户处于不同状态时，系统如何切换不同训练方式
