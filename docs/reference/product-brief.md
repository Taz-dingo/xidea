# 产品简述

## 项目名

- 中文名: 虾学
- 英文代号: Xidea

## 一句话定义

一个把多样输入、学习诊断、多种训练方式和长期记忆整合起来的 AI 学习系统。

当前比赛版主张要再收窄一层：
它首先是一个服务于真实项目学习任务的 AI 学习编排系统，而不是面向所有场景的通用学习助手。

这里的收窄是为了比赛版证明，不代表长期产品边界被缩死。
长期上，Xidea 仍然面向多种项目型学习任务和多种输入、训练形式，只是当前先用一个主案例把核心价值讲透。

## 当前技术方向

- 前端 UI: React + Tailwind CSS + shadcn-friendly structure
- 核心编排: Python + LangChain + LangGraph

这样分层的原因是：

- 前端保持快速表达和高质量 UI 迭代
- 编排核心优先使用更成熟的 Python agent 生态
- 后续可以清晰拆分展示层和智能编排层

## 核心问题

现有产品通常只覆盖学习闭环中的一段：

- Anki 类产品擅长记忆调度，但互动形式单一
- 通用 AI 助手能答疑，但缺少长期记忆状态和学习策略
- 语言学习应用重交互，但多局限于口语或单一学科

Xidea 想解决的是：
用户不仅要“复习”，更要在不同记忆状态和理解深度下，被安排到最合适的学习动作里。
而且这个过程不依赖单一入口或单一训练方式。

当前比赛版只用一个主案例来证明这个命题：
AI 工程师在推进真实项目时，需要真正理解并应用 RAG 系统设计。
这个场景里，用户会经历补充材料、普通问答、概念混淆、方案迁移和后续巩固等连续环节，
比一次性答题或一次性总结更需要系统性编排。

## 产品本质

Xidea 不是某一种学习形式，而是一个 AI 学习编排系统。

它统一管理 5 层能力：

1. 输入形式
   - 同一线程里的普通问答
   - 同一线程里随时补充的原始材料
   - 后续可扩展到图片、音频、视频、网页、PDF、笔记
2. 诊断方式
   - 从历史表现、即时问答、内容理解过程里形成用户画像
3. 训练形式
   - 导师问答
   - 苏格拉底追问
   - 对比辨析
   - 情境模拟
   - 视觉或听觉训练
4. 编排逻辑
   - 判断当前更适合哪种训练动作
5. 记忆回写
   - 把训练结果沉淀成后续学习依据

## 与 Anki 式科学复习的关系

Xidea 不替代科学复习，而是把它放进一个更大的系统里。

关键区分：

- 科学复习回答的是“什么时候该再来复习”
- Xidea 要回答的是“当前这一步到底该怎么学”

因此，科学复习在 Xidea 里更适合作为记忆调度层，而不是产品本体。

系统需要同时维护两类状态：

1. 理解状态
   - 是否真正理解
   - 是否容易混淆
   - 是否能迁移应用
2. 记忆状态
   - 记忆强度如何
   - 上次复习时间
   - 下次建议复习时间

只有当系统判断用户已经基本理解、但需要巩固时，才进入更典型的 spaced repetition 节奏。
如果系统判断用户本质上“没懂”，就应该先教学、解释、对比或练习，而不是直接做卡片式复习。

## 当前判断规则

比赛版默认按下面的顺序理解“现在该怎么学”：

- 没懂: 先教学或澄清
- 容易混淆: 先做对比训练
- 基本理解但不稳: 先做练习或迁移
- 会了但快忘: 再进入复习节奏

这条规则的意义是：
系统先判断学习动作，再决定是否进入科学复习，而不是把所有问题都压成卡片式回顾。

## 核心价值主张

- 支持多样输入，而不是要求用户先手工整理成卡片
- 按记忆状态动态安排复习节奏，而不是固定刷卡
- 按理解薄弱点切换学习方式，而不是统一问答
- 支持文本、图片、音频、视频、公式、网页、PDF 等内容进入同一个学习系统
- 既能学语言，也能学编程、医学、设计、投资等复杂主题
- `flashcards`、quiz、guided QA、study guide 都只是候选学习形式，不是产品本体；系统会判断当前更适合哪一种

但在比赛版表达上，优先只讲下面这句：

- 系统会根据项目上下文和学习状态，决定用户下一步最该怎么学

## 比赛版 Demo 重点

这版 demo 不追求能力全做完，而是清楚展示三个差异点：

1. 用户围绕真实项目进入系统，不是孤立问一个问题就结束
2. 系统会显式维护学习状态，而不是只做即时回答
3. AI 不只是回答问题，而是在输入、诊断、训练之间做编排

第一版默认只证明一次核心判断：
系统能区分用户是没懂、混淆了，还是已经理解但需要巩固。

## 核心体验链路

1. 用户在同一个线程里开始提问，并可随时补充材料
2. 系统理解内容，并同步形成学习者画像
3. 系统识别当前掌握状态、薄弱点和合适的训练切入点
4. 系统按状态选择最合适的训练模式，必要时发起一张或一组轻量学习动作
5. 学习中持续追踪表现并调整后续路径；并不是每一轮都必须出现卡片或同一种形式
6. 学习后沉淀为长期记忆计划和能力画像

## 关键指标

- 首次体验时，用户是否明显感知“这不是普通 AI 问答”
- 单次学习后，用户是否感知理解更深或记忆更稳
- 一周内是否愿意回来继续学
- 不同内容类型进入系统后的转化效率

## 风险

- 如果只是多模态包装，但没有真实的学习策略优势，会变成演示型产品
- 如果诊断不准，路径安排会显得随机
- 如果输入和训练形式很多，但没有统一编排逻辑，产品会显得松散
- 如果把所有问题都交给 spaced repetition，系统会退化回更强包装的复习工具
- 如果内容加工成本过高，通用性会受限
